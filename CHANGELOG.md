# 교육실 개발파트 업무 현황 — 작업 기록

> GitHub Pages + Supabase 기반 단일 HTML 파일(`index.html`) 프로젝트  
> 주소: https://wowowowmj.github.io/studyfile/

---

## 기초 구축

### 초기 대시보드 생성
- 스터디 자료 준비 현황 대시보드 초기 구조 생성
- Supabase 연동: 체크박스·담당자·상세정보 DB 저장 및 페이지 로드 시 불러오기
- async init으로 인한 로드맵·담당자·유형 섹션 0% 표시 문제 수정
- 초기화 오류 시 로딩 오버레이가 사라지지 않는 문제 수정

### Supabase 자료구조 개편
- 기존 분산된 구조 → 한 자료당 완전한 데이터셋을 갖추는 방식으로 전면 개편
- 테이블 구성:
  - `cards` — 자료 메타데이터 (학년, 학기, 유형, 자료명)
  - `card_items` — 항목 목록 (항목명, 순서)
  - `item_states` — 상태 데이터 (완료여부, 상태, 담당자, 마감일, 메모, 링크)
  - `v_full` 뷰 — 세 테이블 조인 (Google Sheets 연동 소스)

## 기타(커스텀) 카드

### Supabase 연동
- 기타 카드를 localStorage 전용 → **Supabase 연동**으로 변경 (크로스 디바이스 동기화)
- `cards`, `card_items` 테이블에 저장
- anon role에 INSERT/DELETE 권한 부여 (`GRANT ALL PRIVILEGES`)
- `v_full` 뷰에 CASE 처리 추가 (grade/semester/type = 'etc'/'custom' → '기타'로 표시)

### 담당자 섹션 반영
- 기타 항목이 담당자별 업무 목록에 표시되도록 `collectAssigneeTasks` 확장

### 삭제 시 부활 현상 수정
- 삭제 후 새로고침하면 항목이 되살아나던 문제 수정
- `deleteItem` → `syncCustomCardToSupabase` 호출로 Supabase 동기화 및 인덱스 재정렬

## Google Calendar 연동

### 기본 캘린더 구현
- ICS 연동 캘린더 추가 (3학년 / 4학년 일정)
- 달력 5열 (월~금) 표시 + 학년 필터 버튼
- 기간형 일정: 시작일만이 아닌 start~end 전체 날짜에 렌더링
- 날짜 칸 클릭 → 일정 목록 팝업 표시
- 모바일 날짜 목록 모달 UX 개선

### ICS 로딩 안정화
- 여러 CORS 프록시 병렬 요청(Promise.any)으로 로딩 속도 개선
- 중복 요청 방지 + 타임아웃 증가 + 프록시 5개 순차 폴백

### 편집 모드 (Google OAuth)
- Google OAuth 로그인 → 일정 추가 / 수정 / 삭제 기능
- 로그인 시 ICS 캐시 제거 후 Calendar API로 실시간 재로드
- 저장 성공 후 해당 날짜의 월로 자동 이동

### 일정 완료 체크
- 날짜 팝업에서 각 일정에 완료 체크박스 추가
- 완료 상태 Supabase(`cal_completed`) 저장으로 다기기 공유
- Supabase Realtime으로 완료 상태 실시간 양방향 동기화
- 완료된 일정: 달력 그리드에서 취소선 표시

### ICS ↔ API 이벤트 ID 통일
- ICS UID는 `이벤트ID@google.com`, API ID는 `이벤트ID` 형식으로 불일치
- 로그인 전 체크한 완료 상태가 로그인 후 초기화되던 문제
- `normalizeCalId()` 함수로 `@google.com` 접미사 제거 후 통일
- 구버전 DB 항목(`@google.com` 포함)도 체크 해제 시 함께 정리

## Google Sheets 연동 (Apps Script)

### 기본 구조
- `전체현황` 탭: Supabase 데이터를 읽기 전용으로 표시
- `상태편집` 탭: 셀 수정 즉시 Supabase에 자동저장 (트리거 방식)

### 주요 컬럼
| 컬럼 | 설명 |
|---|---|
| 학년, 학기 | 회색 (읽기 전용) |
| 자료명, 항목 | 회색 (읽기 전용) |
| 카드ID, 항목번호 | 회색 (읽기 전용) |
| 완료여부, 상태, 담당자, 마감일, 메모, 링크 | 흰색 (직접 편집) |

### 자동화
- 상태 = `완료` 입력 시 완료여부 자동으로 `TRUE` 처리
- 메뉴에서 **선택 행 삭제** 기능 추가

### 한국어 배열 파싱 버그 대응
- Rhino 엔진에서 한국어 문자열 배열 리터럴 파싱 오류
- CSV 문자열 + split() 방식으로 우회 해결

### 주의사항
- onEdit 트리거는 반드시 **설치형 트리거**로 등록해야 Supabase 저장 가능 (단순 트리거는 외부 API 호출 불가)
- 시트에서 수정 후 페이지 확인 시 브라우저 새로고침 필요
- `전체 새로고침`은 Supabase → 시트 방향이므로, 시트 편집 내용을 덮어쓰지 않음

| 항목 | 내용 |
|---|---|
| 프론트엔드 | 단일 HTML 파일 (`index.html`), 바닐라 JS |
